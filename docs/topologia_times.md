## Topologia

| Papel | Responsabilidade |
| --- | --- |
| Time de Produto | Validar ideia, fluxo do cliente, requisitos e apresentacao |
| Time de Aplicacao | Implementar App Blazor, API, banco e testes |
| Time de Operacao | Cuidar de Railway, dominio, pipeline e checklist |
| Time de Qualidade | Verificar build, testes, documentacao, riscos e DoD |

## Modelo de Trabalho

O grupo atua como um time unico de produto/aplicacao, com responsabilidades divididas por tarefa. Para a entrega academica, a comunicacao e centralizada e as mudancas sao publicadas no GitHub principal.

## Interfaces entre Responsabilidades

| Entrada | Responsavel | Saida |
| --- | --- | --- |
| Requisito novo | Produto | Spec atualizada |
| Implementacao | Aplicacao | Codigo funcionando |
| Bug | Qualidade + Aplicacao | Correcao testada |
| Deploy | Operacao | Railway atualizado |
| Avaliacao | Todos | Demonstracao validada |
